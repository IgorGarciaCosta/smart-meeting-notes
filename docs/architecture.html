📋 Componentes

MeetingsController3 endpoints REST — upload, get by id, list all
MeetingProcessingQueueFila in-memory (Channel<Guid>) entre controller e background service
MeetingProcessingServiceBackgroundService que orquestra o pipeline: transcrição → análise → salvar
WhisperServiceChama Python via subprocess — roda Faster Whisper localmente (offline)
GeminiServiceChama Gemini 2.5 Flash via HTTPS — analisa transcript e extrai insights
JsonMeetingStorePersistência simples em arquivos JSON — cada meeting = 1 arquivo

📊 Status Pipeline

0 — UploadedÁudio recebido, aguardando processamento
1 — TranscribingWhisper está transcrevendo o áudio
2 — AnalyzingGemini está analisando o transcript
3 — CompletedTudo pronto — transcript + análise disponíveis
4 — FailedErro em alguma etapa — ver errorMessage
